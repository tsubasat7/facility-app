# 施設内向け ブラウザ判定アプリ 運用マニュアル（完全版）

このアプリは以下の目的で使用します：

・利用者の状態を  
　「安全（clarify）」「注意（doubt）」「要相談（alert）」  
　の3段階で即時判定  
・短期的な感情の急変による誤検知を防止  
・医療行為を行わず、支援判断の補助のみを行う

---

## ✅ 使用開始手順（初回のみ）

1. GitHub にリポジトリを作成
2. `index.html` をそのまま貼り付けて保存
3. `Settings → Pages → main / root` を選択
4. 表示された URL を開く

→ これで即日運用可能

---

## ✅ 日常運用フロー

### 【朝】
・判定画面を開く
・各利用者の数値を入力
・判定結果（色）を確認

| 表示 | 意味 | 対応 |
|------|------|------|
| 緑 clarify | 安定 | 通常対応 |
| 黄 doubt | 注意 | 声かけ強化 |
| 赤 alert | 要相談 | 責任者共有 |

---

## ✅ 誤検知防止ルール（最重要）

・short-window が ±1 を超える場合は **必ず「保留」表示**
・この場合は 30〜60分後に再チェック

---

## ✅ 絶対にやってはいけないこと

・このアプリだけで診断を決めない  
・記録を本人の不利益に使わない  
・判定結果を叱責に使わない  

---

## ✅ 週1メンテナンス

・GitHub のバックアップダウンロード
・ログの傾向確認
・誤判定が連続していないかを確認

---

## ✅ ロールバック方法（事故防止）

1. GitHub → Commits
2. 正常だった日付の commit を選択
3. `Revert`
4. Pages 即時反映

---

## ✅ 法的・倫理的注意

本アプリは：
・診断行為 ❌
・薬剤判断 ❌
・医療指示 ❌

あくまで **支援判断の補助ツール** です。
